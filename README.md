**Разработка интернет-магазина сувенирной продукции **
Данный проект является практической частью магистерской диссертации на тему: «Разработка интернет-магазина сувенирной и бытовой продукции мастеров ГБПОУ РХ «ТКХиС» (Абакан, 2025 г.).
Проект представляет собой функциональное веб-приложение с гибридной архитектурой, предназначенное для автоматизации процессов продажи изделий ручной работы.
Ознакомиться с магистерской работой можно тут https://disk.yandex.ru/i/i0MQ7cLf8LlFJw

**Ключевые особенности**
Гибридная архитектура -> Сочетание классического серверного рендеринга (Flask + Jinja2) для стабильности и SEO с современным SPA-подходом на React для высокоинтерактивных разделов (каталоги, корзина, страницы товаров).
Ролевая модель доступа (RBAC) -> Разграничение прав для трех типов пользователей: Клиент, Продавец, Администратор.
Инженерный подход к БД -> Спроектирована реляционная структура из 11 таблиц, реализованы триггеры на уровне СУБД для обеспечения целостности данных.
Панели управления -> Полноценный Dashboard для продавца со статистикой заказов и CRUD-интерфейсы для администратора.
Адаптивный дизайн -> поддержка мобильных устройств 

**Cтек 🛠 **
Backend: Python, Flask, Flask-SQLAlchemy (ORM), Flask-Login, Flask-Migrate.
Frontend: React, JavaScript (ES6+), CSS3 (Flexbox, Media Queries).
База данных: SQLite (с возможностью легкой миграции на PostgreSQL через SQLAlchemy).
Инструменты проектирования: IDEF0, DFD, ER-моделирование.

**Инструкция по запуску для просмотра**
1. Требования
Python 3.11 или выше
Node.js 14+ (для сборки фронтенда)

3. Настройка бэкенда
3.1 Клонируйте репозиторий.
3.2 Перейдите в папку backend
3.3 Создайте и активируйте виртуальное окружение:

python -m venv venv
venv\Scripts\activate

3.4 Установите зависимости:
pip install -r requirements.txt
3.5 Настройте конфигурацию:
3.5.1Создайте файл .env на основе .env.example.
3.5.2 Сгенерируйте или придумайте SECRET_KEY и укажите путь к БД. # в проекте уже лежит бд для теста с тестовыми данными

4. Запуск
Находясь в папке backend и в запушенном виртуальном окружении или в глобальном, напишите в терминал
python app.py
Приложение будет доступно по адресу: http://localhost:5000

Необходимая фронтэнд часть приложения лежит бандлом в папке backend\static

**Архитектура и проектирование**
В рамках диссертации были разработаны:
Модель TO-BE: Декомпозиция бизнес-процессов в стандарте IDEF0.
Схема потоков данных (DFD): Определение путей движения информации между пользователем и хранилищем.
ER-диаграмма: Спроектирована нормализованная база данных (11 сущностей), исключающая избыточность.
